# 도로 노면 자동화 — 설계 계획 (Phase R)

> KBS TopoMap `RoadConnect`/`RoadFlatten`/`LaneMaker`를 대체하는, **주소 딸깍 → 도로 노면 3D**
> 자동 생성 계획. KBS 로드맵(`kbs_topomap_reference.md` §6) 1순위.
> 작성 2026-07-08. **아직 미구현 — 리뷰·수정용 계획서.**

## 0. 한 줄 요약

수치지형도 **A0010000 도로경계 폴리곤**을 **DEM에 드레이프**해 도로 노면 메시를 만들고(가시 레이어),
선택적으로 **도로 아래 DEM을 평탄화**해 노면이 리플 없이 앉게 한다. 지형·건물과 같은 3소비자
(F2 뷰어 / `.3dm` / SketchUp 확장)로 출력. 오프라인 bake는 DEM과 **같은 SHP 폴더·같은 지역 롤아웃**을
재사용한다.

## 1. 목표 & 산출물

**목표**: 대지 주변 도로를 자동으로 3D 노면으로 재현. 사람 손(도로망 수동 연결·평탄화) 제거.

**산출물(소비자별, 지형·건물과 동일 3경로)**:
- **F2 웹 뷰어**: `geometry.roads` (드레이프된 도로 메시 vertices/triangles) → three.js 렌더(회색 아스팔트 머티리얼).
- **`.3dm`**: `roads` 레이어(중립 회색) 메시.
- **SketchUp 확장(.rbz)**: 백엔드 geometry JSON의 도로 메시를 그룹으로 조립(지형과 동일 패턴).

**단계별 산출 정밀도**:
1. **R1a — 도로 외곽선**: A0010000 경계를 DEM에 드레이프한 `LineLoop`(지적선과 동일 코드 재사용). *가장 싼 첫 가시화.*
2. **R1b — 도로 노면 메시**: 폴리곤 내부를 DEM 해상도로 샘플링·삼각화한 드레이프 면. *진짜 노면.*
3. **R2 — 도로 아래 평탄화**: 종단 프로파일 평활 + 횡단 크라운으로 노면이 지형 리플에 안 흔들리게.
4. **R3 — 보도·차선**: A0033320 보도 폴리곤, A0020000 중심선+`폭원`/`차선수`로 차선 마킹.
5. **R4(별도 Phase) — 수계**: E계열(하천경계 폴리곤 → 표고고정 수면). 로드맵 2순위, 본 문서 밖.

## 3. 설계 분기 & 결정

### 분기 1 — 노면 소스: 오프라인 SHP bake vs VWorld 실시간

| | (A) SHP 오프라인 bake **[추천]** | (B) VWorld 실시간 |
| --- | --- | --- |
| 커버리지 | 베이크한 지역만(DEM과 동일) | 전국 즉시(건물·지적처럼) |
| 품질 | 측량 도로경계(고정밀, 1401/도엽) | **미검증** — VWorld 도로 면 레이어 품질·존재 불확실 |
| 평탄화 결합 | DEM과 같은 파이프라인 → 자연스러움 | 평탄화는 어차피 DEM 수정 필요 → 반쪽 |
| 인프라 | 지역 벡터 파일 서빙(신규, 소형) | 신규 없음(제네릭 클라이언트 재사용) |

**결정: (A) SHP 오프라인 bake.** 근거: ① **도로 아래 평탄화는 DEM(오프라인 산출물) 수정이 필수**라
도로를 DEM 파이프라인에 묶는 게 구조적으로 옳다. ② 노면 폴리곤은 이미 **같은 SHP 폴더**에 있음
(A0010000 = 등고선 F0010000과 동일 폴더). ③ VWorld 도로 면 품질 미검증. **단, 착수 전 확인 스텝**:
VWorld `req/data`에 도로경계 면 데이터셋이 있고 품질이 충분하면 R1(가시 노면)만 (B)로 전환 가능
(평탄화는 여전히 (A)). → **§7 확인 스텝**.

### 분기 2 — 평탄화 위치: bake-time DEM 굽기 vs runtime 클립 후

| | bake-time(DEM에 소각) | runtime(클립 후 평탄화) **[추천 시작점]** |
| --- | --- | --- |
| 일관성 | 지형·노면 항상 일치 | 토글 가능(도로 on/off) |
| 비용 | bake 1회 | 쿼리마다(작음, 국소) |
| 되돌리기 | 재베이크 필요 | 자유 |

**결정: R2는 runtime 평탄화로 시작**(토글·반복개발 유리), 안정화되면 bake-time 소각을 옵션으로.

### 분기 3 — 서빙: 지역 벡터 파일

DEM은 COG 타일 + `/vsicurl` 윈도우 읽기. 도로 폴리곤(벡터)은 **지역별 GeoPackage(.gpkg) 1개**를
GCS에 두고 manifest가 참조, 런타임이 해당 지역 파일을 받아 bbox로 클립(대전 도엽 1401폴리곤 ≈ 수백 KB,
가볍다). manifest는 DEM처럼 git 추적, .gpkg는 미추적(GCS 서빙).

## 4. 아키텍처 (모듈 & 데이터 흐름)

```text
[bake, 오프라인]
  src/terrain/road_bake.py   ← 신규
    A0010000(+A0033320) SHP 읽기 → 도엽 dedup·5187→5186 재투영(contour_bake 헬퍼 재사용)
    → 정리(무효/미세 폴리곤 제거, 중복 용접) → 지역 GeoPackage 저장 + manifest 갱신
    (교량 A0070000은 'bridge' 플래그로 분리 저장)

[runtime, pipeline.generate]
  src/terrain/store.py       find_road_file(bbox)         ← manifest에서 지역 도로 gpkg 찾기
  src/geometry/road.py       ← 신규
    clip_roads(gpkg, bbox) → 폴리곤들
    drape_polygon(poly, dem, cell) → RoadMesh(vertices,triangles)  # 내부 샘플링·삼각화·z=elev_at
    (R2) flatten_under_roads(dem, polys) → DEM 국소 평탄화
  src/pipeline.py            layers={"roads": True} 분기 → RoadMesh 생성 → 출력 3경로에 전달

[출력]
  src/pipeline.py::_build_geometry   + "roads": {vertices, triangles}         (F2 뷰어)
  src/output/rhino.py                + 'roads' 레이어 Mesh                    (.3dm)
  src/output/skp_mcp.py              + 도로 그룹 (지형 메시와 동일 방식)      (.skp/확장)
  frontend/src/Viewer3D.tsx          buildRoads() 회색 메시 + 표시 토글
```

**재사용**: `contour_bake._find_shp/_sheet_key/_to_target_crs`(SHP 읽기·dedup·재투영),
`dem.DEMPatch.elev_at`(드레이프 z), `terrain_mesh`(메시 자료구조), `_build_geometry`/`Viewer3D`
(지적·지형에 이미 있는 드레이프·토글 패턴).

**핵심 신규 컴포넌트 = `drape_polygon`(드레이프 메시 생성기)**: 폴리곤 하나를 DEM 위 삼각 메시로.
- 폴리곤 내부에 DEM 셀 간격(≈2.5–5m) 격자점 생성 + 경계 정점 → 폴리곤으로 클립.
- 제약 삼각화(구멍=중앙분리대 지원): `shapely`로 격자·경계 구성 후 삼각화. 라이브러리 후보:
  `mapbox_earcut`/`triangle`(미설치) 대신 **경계+내부격자 Delaunay 후 폴리곤 밖 삼각형 제거**
  (scipy `Delaunay`만으로, 이미 의존성). 오목·구멍은 삼각형 중심점 point-in-polygon으로 컬링.
- 각 정점 z = `dem.elev_at(x,y)` (+R2에선 평탄화된 z).

## 5. 정확도 계획 — "정확하고 잘 나오게"

도로가 어색해지는 원인과 대책:

1. **횡단 기울어짐/리플(R1 드레이프의 한계)**: 5m DEM은 도로를 모르므로 폭 방향으로 노면이
   기울거나 물결친다. → **R2 평탄화 필수.**
2. **R2 평탄화 알고리즘**:
   - **종단 프로파일**: 도로 중심선(A0020000)을 따라 DEM 표고를 샘플 → 이동평균/스플라인 평활 →
     끊김 없는 종단 곡선.
   - **횡단 크라운**: 중심선에서 가장자리로 완만한 배수구배(예 2%)만 주고 그 외 평탄.
   - 노면 정점 z = 종단프로파일(중심선 투영 거리) − 크라운(중심선 수직거리). 리플 제거.
3. **교량·고가·입체(A0070000 교량 / A0090000 고가차도·지하차도 / A0110020 터널)**: 지형 평탄화
   **금지**(공중/지하 구조물). **데크 절대높이는 SHP에 없다**(A0070000 높이필드 없음, A0090000
   `높이`는 "통과높이=밑 여유고"이며 고가차도는 빈값) → **DSM 드레이프로 실측 데크높이·경사** 획득이
   정답. DSM 없으면 abutment 보간+`구분` 분류로 근사(T0). 분류·처리 상세는
   [dem_dsm_strategy.md](dem_dsm_strategy.md) §5. R1(MVP)은 표면도로만, 입체는 제외+경고.
4. **절토/성토 불연속**: 평탄화하면 도로변과 주변 지형 사이 단차 발생 → **스커트(경사면) 생성**
   또는 평탄화 폭을 도로+여유로 페더링. KBS `makeskirt` 대응. R2 후반.
5. **z-fighting**: R1은 노면을 지형 위로 살짝(+0.1m) 리프트(지적선과 동일). R2 평탄화는 노면이
   곧 지형이 되므로 z-fighting 소멸(더 나은 해법).
6. **이음매/중복 폴리곤**: 인접 도엽·교차부에서 폴리곤이 겹침 → bake 시 `unary_union`으로 용접 후
   재분해, 또는 런타임에서 겹쳐도 무해(같은 회색). 슬리버 폴리곤(면적<임계) 제거.
7. **구멍(중앙분리대·교통섬)**: 폴리곤 interior ring 지원 — 삼각화에서 컬링.
8. **DEM 없는 지역**: 도로는 DEM 드레이프 의존 → DEM 없으면 조용히 생략(+warnings), 지형과 동일 정책.

**검증(자동 QA, 로드맵 §6B 씨앗)**: 생성 후 자동 점검 — 노면 삼각형 최대 경사 임계 초과(리플 잔존),
도로-지형 단차 과대, 폴리곤 대비 메시 면적 누락률. 실패 시 warnings. → 향후 "자동 QA 레이어"로 성장.

## 6. KBS TopoMap 대조

KBS 도로 모듈(§2): `RoadConnect`(중심선 연결·용접) · `RoadFlatten`(도로 아래 평탄화) ·
`LaneMaker`/`ParkingLine`(차선·주차선). (핵심 로직 `.rbe` 암호화 — **미열람**, 동작·공개규격만 참조.)

| 항목 | KBS | arch-site-model(본 계획) |
| --- | --- | --- |
| 노면 소스 | 사람이 SHP 폴더·bbox 지정, 중심선 수동 연결 | **주소→자동** bbox, A0010000 폴리곤 자동 |
| 노면 생성 | `RoadConnect`로 중심선 용접 후 면화(수동 개입) | 폴리곤 직접 드레이프(용접 자동 `unary_union`) |
| 평탄화 | `RoadFlatten` 수동 영역 지정 | 중심선 종단 평활+크라운 **자동** |
| 지형 품질 | SketchUp서 등고선→TIN → 나쁜 삼각형 → 수동 수리 | **격자 DEM**이라 애초에 깨끗 → 수리 불필요 |
| 차선 | `LaneMaker` 수동 스타일(길이·간격·색) 지정 | A0020000 `폭원`/`차선수`에서 규칙 자동(R3) |
| 배포 | 각 PC 수동 설치 | 서버 + 웹/`.3dm`/확장 |

**같은 점**: 근본 연산(도로 폴리곤을 지형에 드레이프 + 도로 밑 평탄화)은 동일. 데이터도 동일 수치지형도.

**다른 점 / 우위**: ① **자동**(주소·연결·평탄화·차선 규칙화) vs KBS 수동. ② **격자 DEM**이라 KBS의
수동 메쉬 수리(FlipEdge 등 ~10개 툴) 자체가 불필요. ③ 서버·웹 배포.

**KBS가 나은 점 / 우리 한계**:
- KBS는 사람이 눈으로 보정 → **엣지 케이스(복잡 교차로·다층 입체)에서 수동이 더 정확**할 수 있음.
  우리는 자동이라 이상 케이스에서 어색할 수 있음(→ 자동 QA로 보완).
- KBS는 차선 스타일을 세밀히 손댐. 우리 R3 자동은 표준 규칙만.

**한계(공통·구조적)**:
- 도로경계는 **2D**(표고 없음) → z는 DEM 드레이프에 의존 → **5m DEM 정밀도가 상한**.
- **지역 한정**(DEM처럼 베이크한 지역만) — 전국 즉시 아님.
- **입체·터널·복층 도로**는 2D 폴리곤에 안 담김 → 교량만 부분 대응, 나머지 한계.
- DEM은 등고선 기반(도로 무시)으로 구워짐 → 평탄화는 근사.
- 수치지도 자체 오류(끊긴 도로망·겹친 폴리곤) → 정리 로직으로 완화하되 완전 제거 불가.

## 7. 단계별 로드맵 & 착수 순서

- **R0 — 확인 스텝(반나절)**: (a) VWorld 도로 면 데이터셋 존재·품질 프로브(있으면 R1 소스 재고),
  (b) `drape_polygon` 삼각화 방식 PoC(scipy Delaunay+컬링 vs 라이브러리), (c) A0010000 폴리곤
  대전 실데이터로 드레이프 1장 렌더 확인.
- **R1a — 도로 외곽선 ✅완료(2026-07-08)**: `road_bake`(A0010000→지역 GeoJSON+manifest) +
  `road.clip_roads`(json+shapely) + `store.find_road_file` + `pipeline layers.roads` →
