# KBS TopoMap 기능 대조 & 로드맵 (레퍼런스)

> arch-site-model이 자동화하려는 원본 워크플로우인 **KBS TopoMap 3.9.2**(수동 SketchUp 확장)의
> 전체 기능을 정리하고, 우리 프로젝트와 대조해 "무인·정확·처음부터 완결"로 가는 로드맵을 남긴다.
>
> **작성 근거**: KBS 배포본에서 **읽을 수 있는 것만** 사용했다 — 파일명(21개 모듈), 아이콘, UI HTML
> (`topomap.html`·`mapcode.html`·`laneoption.html` 등), `*.ini` 포맷, `notice.txt`. 핵심 로직
> `.rbe`는 **암호화된 SketchUp 루비**라 열지 않았고, 열어서도 안 된다(아래 §7 법적 주의).

## 2. 전체 기능 인벤토리 (21개 모듈)

| 그룹 | 모듈 / 툴 | 하는 일 |
| --- | --- | --- |
| **데이터** | `TopoMap_data` | 수치지형도 SHP 로드·파싱(538종 표준코드 전 계열), 메뉴 `TopoMap_menu`, 공용 `Lib` |
| **지형 생성** | `Cont_Terrain` · `Cont_Polygon` · `Cont_Repair` | 등고선+표고점 → TIN 지형, 등고선 폴리곤화, 등고선 오류 수리 |
| | `makeskirt` · `stepwise`(계단) | 지형 가장자리 스커트, 계단식 표현 |
| **메쉬 수리 (수동)** | `FlipEdge` · `FlipFaceEdge` | 삼각형 대각선 뒤집기(평평·꼬임 삼각형 개선) |
| | `vertex` · `vtoz` · `flatten` | 정점 편집, 정점 표고 지정, 영역 평탄화 |
| | `clean` · `reverse` · `copyslope` · `drop` · `reconstrusion` | 정리, 면 뒤집기, 경사 복사, 지형 투영, 재구성 |
| | `groupunion` · `MoveToGroup` · `pushpull` | 그룹 불리언 합집합, 그룹 이동, 돌출 |
| **드레이프** | `FastDrape` (Superdrape 파생) | 항공/정사영상·선형을 지형에 투영 |
| **도로 (최대 모듈)** | `RoadConnect` · `RoadFlatten` | 도로 중심선 연결·용접, 도로 아래 지형 평탄화 |
| | `LaneMaker` · `ParkingLine` | 차선(Dash/Single/Double: 길이·간격·폭·깊이·색), 주차선 3D 마킹 |
| **UI** | `progressbar` · `config` · `updateui` | 진행바, 설정, UI 갱신 |

**관찰**: 21개 중 **약 10개(FlipEdge·Cont_Repair·vertex·vtoz·flatten·clean·copyslope·reconstrusion·
makeskirt·stepwise)가 "수동 메쉬 수리" 툴**이다. 이유 = KBS가 등고선→TIN을 SketchUp에서 직접
만들어 나쁜 삼각형(평평·꼬임·구멍)이 생기고, 그걸 사람이 손으로 고쳐야 하기 때문. → **§4 핵심 통찰**.

## 3. 수치지형도 표준코드 체계 (mapcode.html, 국토지리정보원 공개 규격)

KBS는 참조용으로 **538종 표준코드 전체**를 임베드하고 있다(우리도 동일 SHP를 읽으므로 그대로 유효).

| 대분류 | 계열 | 종수 | 예시 코드 |
| --- | --- | --- | --- |
| A | 도로·교통 | 105 | 도로경계 `A0010000`, 인도 `A0033320`, 철도 `A0150000` |
| B | 건물 | 145 | 건물경계 `B0010000` |
| C | 시설물(댐·제방·교량·옹벽) | 209 | 댐 `C0010000`, 제방 `C0050000`, 잔교 `C0080000` |
| D | 식생·경작 | 30 | 경지계 `D0010000` |
| E | 수계(하천·해안·호소) | 21 | 하천경계 `E0010001`, 해안선 `E0080000` |
| F | 지형(등고선·표고) | 28 | **등고선 `F0010000`**, **표고점 `F0020000`** |

> arch-site-model은 현재 **F0010000(등고선) + F0020000(표고점)만** 사용(지형). 건물은 SHP(B계열)
> 대신 VWorld API로 취득. A·C·E 계열은 미사용 → §5 갭.

## 4. 핵심 통찰 — "KBS를 그대로 따라가면 퇴보"

**arch-site-model은 지형을 오프라인 격자 DEM(guarded CloughTocher)으로 굽고 런타임에 깨끗한 TIN을
만든다.** 그 결과 **KBS의 수동 메쉬 수리 툴 ~10개가 우리 구조에선 아예 불필요**하다 — 나쁜 삼각형이
애초에 안 생기므로 FlipEdge·Repair·vertex·flatten 등을 사람이 돌릴 일이 없다.

즉 "사람 손 안 거침"의 절반을 **구조로 이미 달성**했다. 이 수리 툴들을 재구현하는 것은 오히려 퇴보다.

추가 우위:
- **건물**: KBS = 수치지도 2D footprint(층수 없음). 우리 = **VWorld 실층수(gro_flo_co)** → 실제 높이.
- **주소**: KBS = 없음. 우리 = VWorld 지오코딩(지번+도로명) → "주소 딸깍".
- **배포**: KBS = 각 PC 수동 설치. 우리 = 서버(Cloud Run) + 웹/`.3dm` + 확장.

## 6. 로드맵 — "무인·정확·처음부터 완결"

**완전 무인의 진짜 장벽은 기능이 아니라 두 가지다:**

- **(A) 데이터 품질** — 수치지도엔 오류가 있다(안 닫힌 등고선, 끊긴 도로망, 겹친 폴리곤). KBS는 이걸
  **사람이 수동 툴로** 고친다. 우리가 무인이 되려면 이 수리를 **자동화**해야 한다(등고선 자동 폐합,
  도로망 자동 스냅, 토폴로지 자동 검증). **여기가 KBS를 앞설 진짜 엔지니어링 지점.**
- **(B) 검증 자동화** — "처음부터 정확"은 생성 후 **자동 QA**(건물 지면 침몰·지형 스파이크·도로
  연결성 검사)로 담보된다. 사람 눈검사를 코드가 대체.

우선순위:

1. **도로 노면 자동화** ⭐ (최대 가치·데이터 보유): 수치지도 A계열 도로경계/중심선 → 노면 폴리곤을
   DEM에 드레이프 + 도로 아래 평탄화. `contour_bake`와 같은 오프라인 굽기 패턴 재사용.
2. **수계 자동화**: E계열 하천/해안 폴리곤 → 표고 고정 수면(평면).
3. **자동 수리·QA 레이어**: 등고선 폐합·토폴로지·생성물 검증 (무인화의 핵심).
4. **차선/주차선**: A계열 선형에서 규칙 기반 자동 생성(KBS는 수동 스타일 지정).
5. **교량·주요 시설**(C): 선택.

결과적으로 arch-site-model은 KBS의 **상위집합**이 된다:
> KBS(수동 수리 다수) < arch-site-model(격자DEM으로 수리 불필요 + 주소 자동 + 실층수 +
> 자동 도로/수계 + 자동 QA).

### 성능 주의 (SketchUp 확장 경로)

밀집·대반경 사이트(예: 신반포 반경 250m = 건물 386개)는 SketchUp 확장이 **건물마다 `pushpull`**을
돌려 렉·크래시가 난다. 대안:
- **웹 미리보기(F2)·`.3dm` 다운로드는 SketchUp 불필요** — 서버가 지오메트리를 만들어 렉이 없다.
  밀집지는 이 경로 권장.
- SketchUp 확장을 쓰려면 (a) 반경 축소, (b) `builder.rb`를 pushpull 대신 직접 메쉬 조립 +
  루프 내 `grep` 제거로 최적화(미착수).

## 7. 법적 주의 (.rbe)

- `.rbe`는 SketchUp 공식 **암호화 포맷 = 저작권 기술적 보호조치(TPM)**. KBS는 "All rights reserved"
  상용 저작물이다.
- **복호화·역어셈블 금지**: TPM 우회는 저작권법 §104-2 / DMCA §1201 이슈. 목적이 "코드 복제"면 복제
  침해까지 붙는다.
- **우리 방식은 clean-room 재구현** — 동작·데이터 포맷·공개 규격만 보고 밑바닥부터 독립 구현. 이는
  합법·안전하며, 암호화 소스를 읽으면 이 방어막이 깨지므로 **읽지 않는 것이 원칙이자 이득**.
- 이 문서의 모든 정보는 **읽을 수 있는 파일 + 공개 규격**에서만 얻었다.
