# DEM + DSM 이원 전략 — 지면은 DEM, 공중 구조물은 DSM

> 정확도·실사성을 끌어올리기 위해 **수치표고모델(DEM, 맨땅)**과 **수치표면모델(DSM, 표면)**을
> 역할별로 나눠 쓰는 전략. 도로(고가/교량), 건물 높이, 지형에 걸쳐 적용. 작성 2026-07-08.
> 도로 상세는 [road_surface_plan.md](road_surface_plan.md), 배경은 [[nationwide-dem-ngii-source]].

---

## 0. 한 줄 원리

**지면에 닿는 것 = DEM(맨땅). 공중에 뜬 구조물 = DSM(표면).**
DEM은 등고선이 못 담는 교량·고가 상판을 모르고, DSM은 라이다가 그 상판을 실측한다. 둘은 경쟁이 아니라 보완.

---

## 1. R0 조사 결과 (2026-07-08, NGII)

- **DSM 실재**: NGII가 **항공레이저측량(LiDAR)으로 수치표면모델(DSM) 구축** 중(명시 확인).
- **해상도·커버리지**: **1m 라이다 = 전국 도시지역**(서울·경기·6대 광역시 — 우리 현 커버리지와 겹침),
  5m = 남한(공개 여부 논란), 90m/10m = 전역 공개.
- **라이선스**: **공공누리(KOGL) 제1유형** — 출처표시 조건 **상업적 이용 포함 자유이용**(정사영상과 동일 근거).
- **포맷/좌표계**: **IMG·XYZ, TM중부(EPSG:5186)** → 파이프라인 직접 호환(rasterio가 IMG 읽음).
- **접근 — 블로커(2026-07-08 정찰 확정)**: 고해상도 DSM은 민간 개발 경로로 **취득 불가**.
  - NGII 항공라이다/DSM 원시점군: **공문(기관) 신청자에 한해** 온라인 제공 → 개인/소규모 불가.
  - 지자체 DSM(인천·제주 등, 삼아항업/산림빅데이터): **₩10,000,000 + "민간 대상 공개제한자료로
    민간 제공 불가"**(공공기관·지자체 한정). 대전 미확인.
  - 무료 글로벌 DSM(Copernicus GLO-30·AW3D30): **30m** → 폭~20m 데크·개별 건물을 한 픽셀이 뭉갬 →
    **용도상 무용**. Copernicus 10m는 유럽만.
  - 서울 S-MAP: 서울 전용 3D 모델(래스터 DSM 아님), 대전 없음.
  - 국토정보플랫폼 자료실 셀프 다운로드: **공개DEM(거친 것)만**, DSM 없음.

**결론(수정): DSM-드레이프 realism(T1)은 민간 경로에선 블로커.** 원리는 유효하나 데이터가 막힘.
→ **T0(휴리스틱 데크: abutment 보간 + 통과높이 + `구분` 분류)로 착수**, DSM은 **기관 접근/데이터
협약이 생기면 승격**(정사영상·setback과 같은 "블로커 대기" 항목으로 관리). 표면 도로·건물 높이는
DSM 없이도 이미 실제(DEM 드레이프 / VWorld 실층수)라 앱 가치는 유지된다.

## 2. 역할 배정 — 피처별 DEM vs DSM

| 피처 | 필요 표고 | 소스 | 비고 |
| --- | --- | --- | --- |
| 지형(terrain) | 맨땅 | **DEM** ✓(현행) | DSM 쓰면 나무·건물 혹 섞임 → 금지 |
| 건물 앉힘(base_z) | 맨땅 | **DEM** ✓(현행) | 변경 없음 |
| 표면 도로 | 맨땅 | **DEM** ✓ | 이미 실제(드레이프) |
| **고가/교량 데크** | 구조물 상판 | **DSM** ⭐ | 라이다 실측 → 드레이프 시 실제 높이·경사 자동 |
| 수계 수면 | 맨땅(제방/수위) | **DEM** ✓ | 물은 라이다 반사 안 됨 |
| 정사영상 | 지형 | **DEM** ✓ | 변경 없음 |
| **건물 높이 검증** | 지붕 | **DSM(측정)** | §4 — 누락 층수 보정·QA |

---

## 3. 남은 작업(도로)에 적용

- **표면 도로(A0010000)** → DEM 드레이프(이미 실제).
- **고가/교량 데크** → **DSM 드레이프**. `구분`으로 분류된 데크 폴리곤을 DEM 대신 DSM에서 샘플 →
  실측 높이·종단경사 자동(별도 abutment 보간·통과높이 추정 불필요). road_plan의 realism T1이
  "드레이프 소스만 DSM으로 교체"로 단순해짐.
- **지하차도/터널** → DSM에도 안 나옴 → 생략/포털 유지.
- **복층 3+** → DSM이 각 상판을 다른 높이로 실측 → 자동 재구성 가능성 상승(2D+휴리스틱보다 우수).
- **수계** → DEM.

## 4. 기존 로직 retrofit — DSM/라이다로 개선

| 기존 | DSM/라이다 적용 | 원칙(재현≠상상) |
| --- | --- | --- |
| **건물 높이** = VWorld 층수×3m; 누락 시 기본 1층(추측) | **누락 건물** = DSM지붕−DEM지면 = 실측 높이로 대체. 전건물 VWorld 대비 **불일치 자동 QA 플래그** | ✅ DSM은 **측정**이지 AI 추정 아님 → 원칙 강화 |
| **지형 계단현상**(열린 TODO) | **1m 라이다 DEM**은 등고선 양자화가 없어 **계단현상 소멸** → 격자 솔버 TODO를 데이터로 해결 | ✅ |
| 건물 앉힘 | 변경 없음(DEM) | — |
| 정사영상 | 변경 없음 | — |
| **DEM 소스**(현 등고선 bake 5m) | 라이다 1m DEM(IMG)로 교체 시 `contour_bake` 생략·정밀↑([[nationwide-dem-ngii-source]] 예고) | 선택(지역별) |

**건물 높이가 최대 수혜**: 층수 누락 건물이 "기본 1층" 뭉갬 → **측정 높이**로 정상화 + 전건물 교차검증.
단, DSM은 옥탑·펜트하우스 포함 → **검증/보정용**(VWorld 층수 대체 아님).

## 5. 입체 구조물 필드 정정 (2026-07-08 실측)

> road_plan 초안의 "4.5 = 통과높이" 뉘앙스를 실 필드명으로 정정한다.

대전 도엽 DBF 실측(EUC-KR):

| 레이어 | 구조물 | 실제 필드 | 데크 절대높이? |
| --- | --- | --- | --- |
| A0070000 | 교량(계룡대교, 갑천 위) | 명칭·종류(도로교)·연장·폭·재질·**하천명** | ❌ 높이 필드 **없음** |
| A0090000 | 고가/지하차도 | 명칭·**구분(고가차도/지하차도)**·연장·**높이**·통과하중·보도 | △ 아래 |
| A0110020 | 터널(도솔터널) | 명칭·연장·폭·**높이(4.5)** | 터널 통과높이 |
| A0063321 | 보도육교 | 명칭·연장·폭·**높이**·형태(장애인이용) | 육교 아래 통과높이 |

**"높이"의 정체**: 구조물 **아래/안의 여유높이(통과높이·유효고)** — "밑으로 5m 차 통과". **데크 상판의
절대표고가 아님.** 게다가 정작 **고가차도(괴정육교)는 높이=0.0(빈값)**. "통과하중"은 하중(톤).
→ **데크 절대표고는 데이터에 없음 → DSM 필수.** 단 **`구분` 필드로 고가/지하/터널 분류는 정확**.

**클래스별 처리**:
- 표면 도로 → DEM 드레이프.
- 고가차도·교량 → **DSM 드레이프**(실측 데크). DSM 없으면 abutment 보간+`구분`으로 근사(T0).
- 지하차도·터널 → 매설부 생략, `높이`로 포털 근사.
- 복층/애매 → 자동 QA 플래그(가짜 생성 금지).

## 6. 인프라 — DSM = "두 번째 래스터 레이어"

DEM 파이프라인을 **그대로 미러링**(엔진 구조 재사용):
```text
DSM IMG 취득(지역별, 사람 손) → scripts/dsm_to_cog.py(= dem_to_cog 복제) → GCS 업로드
manifest.json에 dsm 타일 필드 추가(DEM과 나란히)
config.DSM_TILE_BASE(/vsicurl) — DEM_TILE_BASE와 동형
런타임: clip_dsm_mosaic(= clip_dem_mosaic 재사용) → 고가/건물 폴리곤에만 국소 샘플
```
- **재사용**: 클립·모자이크·표고보간(`dem.py`), COG 변환·업로드 스크립트, store/manifest 조회.
- **신규(소량)**: manifest dsm 필드, `config.DSM_TILE_BASE`/`dsm_tile_path`, `find_dsm_tiles`.
- **비용**: 지역당 래스터 2겹(DEM+DSM) → 저장·취득 노력 2배, **베이크한 지역만**(전국 즉시 아님).

## 7. 데이터 취득 (사용자 스텝)

1. **포털 확인(먼저)**: `map.ngii.go.kr` 로그인 → 대전에 **1m 라이다 DTM(DEM)·DSM** 제공/신청절차 확인.
2. **대전 1지역 테스트 다운로드**(전국 아님): 이미 DEM 있는 대전만 받아 "고가 데크가 DSM으로 실제처럼
   뜨는가" 실증. 신반포/계룡대교 등 교량 포함 영역 권장.
3. 검증되면 6개 광역단체로 확대(DEM 롤아웃과 동일 반복).

**라이선스**: KOGL-1 → 상업적 배포 가능, **출처표시**(정사영상 provenance에 함께 표기).

## 8. 통합 로드맵

1. **R0 확정**: (사용자) 포털에서 대전 1m DTM/DSM 접근 확인. ← **지금 여기**

4. **건물 높이 retrofit**: 누락층수 DSM 실측 + 전건물 QA 교차검증.
5. **(보너스) 라이다 DEM**으로 계단현상 TODO 해소 검토.
6. 수계(별도 Phase R4) = DEM.

## 9. 정직한 한계

- **DSM 노이즈**: 나무·옥탑 포함 → 폴리곤 마스킹 국소 샘플, 그래도 건물 높이는 검증용(대체 아님).
- **물·터널**: DSM 무의미 → DEM/생략.
- **해상도 상한**: 1m는 좁은 육교 놓칠 수 있음(더 세밀은 용량↑).
- **접근**: 포털 로그인·신청절차·대용량 S/W → **자동화 불가, 사람 손**. 지역 한정.
- **설계 디테일**(데크 두께·교각·편경사): 어떤 공개 GIS에도 드묾 → 표준값 근사(realism 최종 상한).
